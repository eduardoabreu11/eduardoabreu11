## 🍽️ MenuFlow API

Backend do **MenuFlow**, um **SaaS de cardápio digital para restaurantes**, desenvolvido para dar suporte ao **cardápio público**, **painel do restaurante** e **painel master da plataforma**.

### Estrutura do sistema

* **Cardápio público** acessado pelo cliente final via slug do restaurante
* **Painel do restaurante** para gerenciamento do cardápio e operação do negócio
* **Painel master** para controle geral da plataforma, restaurantes e planos

### Funcionalidades implementadas / previstas

* autenticação com JWT
* middleware de autenticação e roles
* proteção multi-tenant entre MASTER e RESTAURANT_OWNER
* CRUD de restaurantes
* CRUD de categorias
* CRUD de produtos
* CRUD de banners
* CRUD de planos
* dashboard do restaurante
* cardápio público por slug
* validação com Zod
* tratamento global de erros
* proteção contra JSON inválido
* rate limit no login
* headers de segurança com Helmet
* bloqueio de acesso entre restaurantes diferentes
* bloqueio de restaurante inativo no cardápio público

### Arquitetura

Organizado em camadas com:

* `controllers`
* `services`
* `repositories`
* `middlewares`
* `validations`
* `routes`

### Stack utilizada

`Node.js` `Express` `TypeScript` `PostgreSQL` `pg` `JWT` `bcrypt` `Zod` `Helmet` `CORS` `express-rate-limit`

🔗 **Repositório:** [MenuFlow API](https://github.com/eduardoabreu11/menuflow-api)

## 🖥️ MenuFlow Web

Frontend do **MenuFlow**, responsável pela interface visual da plataforma, conectando o usuário à API do sistema.

O projeto foi pensado para funcionar em **três áreas principais**:

* **cardápio público**
* **painel administrativo do restaurante**
* **painel master**

### Funcionalidades / estrutura

* cardápio público por slug do restaurante
* dashboard administrativo
* gerenciamento de categorias
* gerenciamento de produtos
* gerenciamento de banners
* configuração do restaurante
* link público do cardápio
* integração com autenticação e serviços da API
* interface responsiva com foco em usabilidade

### Stack utilizada

`Next.js` `React` `TypeScript` `Tailwind CSS` `shadcn/ui` `Radix UI` `Lucide React` `JWT`

### Destaques técnicos

* estrutura com **App Router**
* componentes reutilizáveis
* services centralizados para integração com backend
* organização por áreas do sistema
* frontend preparado para evolução do SaaS

🔗 **Repositório:** [MenuFlow Web](https://github.com/eduardoabreu11/menuflow-web)
